Join Partial Entries

Combine two parts of entry files that span sections.

When we convert page files into entry files, we often lose the last part of any entry that occurs on the final page of a section, unless it happens to end at the end of that page. Normally we have to recombine the two parts of the entry: the first part at the end of a section, and the second part from the beginning of the next section.

Note: This procedure assumes that a temporary title was added to the second part of the entry before the page files were converted into entry files. See Add Titles to Partial Entries.
  1. In Oxygen XML Editor, open the file containing the beginning of the entry. This is the file we will keep. Normally, this is the last file in a section.
  2. Open the file containing the end of the entry. Normally, this is the first file in the next section. It will have a temporary <label> tag with the correct entry term, followed by "CONTD."
  3. Select everything from the first <div> tag to the closing <div> at the end of the file, and copy CTL+C it to the clipboard .
  4. In the file containing the beginning of the entry, scroll to the last </p> tag at the end of the text, where the remainder of the text should go. Make a note of the line number.
  5. Paste CTL+V the ending of the entry into position.
  6. Scroll back to the line number where you added the text. We modify the contents of the <div> tag, to convert it into a normal <pb> tag. 
     <div xmlns="http://www.tei-c.org/ns/1.0" xml:id="eb09-21-s01-0122-02"
                    facs="encyclopediabrit21newyrich_raw_0140.jp2" type="entry">
    1. Change <div> to <pb>.
    2. Remove both the @xmlns from the beginning of the tag and the @type from the end.
    3. Remove the last two digits and the hyphen from the end of the @xml:id.
    4. Leave alone the filename in @facs. 
    <div xml:id="eb09-21-s01-0122" facs="encyclopediabrit21newyrich_raw_0140.jp2">
  7. Go the end of the file and correct any unneeded closing <div> tags and then validate the file.
  8. At this point, you can delete the file containing the end of the text. In the entry-inventory, delete the filename for the deleted file, but leave the other data for it in place. Add a note in the comment column that the partial entry file has been corrected and the file deleted. Make a note in the entry-inventory for the beginning of the entry that the file has been corrected.
  9. If there are other entry files with the same page number as the deleted file, they will have to be renamed. The temporary file will be marked as the first entry on the page, but that is inconsistent, since we always number entries as the first entry to start on the page.
