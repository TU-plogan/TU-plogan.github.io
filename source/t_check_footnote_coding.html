Check footnote coding

Use a simple search to check if you have the right number of note codes on all pages.

Every page with footnotes must have a footnote anchor for every block of footnote text. If there are three footnote anchors, there should be three footnotes at the end of the file. In Oxygen XML Editor, you can easily double-check the notes for consistency, using Find/Replace in Files.

  1. Open the XML-project.
  2. In the Project window, right-click the 2-pages-htmlfolder containing your files.
  3. Select Find/Replace in Files in the context menu.
  4. The Find/Replace in Files window opens. Enter @@ in the Text to find field and click Find all at the bottom of the window.

  5. The matches appear in the Results View, with the search term highlighted.
  6. Look at the example above of a page with one footnote anchor and the corresponding footnote text. This is typical for every page with one footnote. Why does the footnote text, with three "@" symbols, appears twice? Because we are searching for two "@" symbols, rather than three. The search counts both the first two and the second two as valid results.
  7. Now look at the line with the file information. It lists the results as "(3 items)" in that file. Every file with one footnote will produce three items. Files with two notes produce six items. If the notes were flagged correctly, the number of items in every file will always be a multiple of three.
  8. Collapse the list of results by right-clicking and selecting Collapse All. Review the items number for each file. Open any file with an incorrect number of items and inspect it manually, to fix the results.
  9. Make a note in the inventory file for the section indicating a problem with footnote coding on the page. This way, if the HTMLhas to be output again for any reason, the correction can be made in the OCR-Project file.
