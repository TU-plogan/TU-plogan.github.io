Fix Runovers

Fix runover entries from for files and notes.

Runovers appear in two places:
  1. entries that have to be pieced back together.
  2. footnotes that run over to subsequent pages.
This section details how to rejoin entry pieces. For footnote runovers, see Review Inventory Comments.
  1. In the entry-inventory, see if the entry term for the first entry in the section ends with "CONTD". This indicates that it contains continuation text that runsover from the last entry in the previous section.
  2. Open both files: the original entry file (called an 'entry stub') and the continuation file.
  3. In the continuation file, copy all content from the <div>at the start to the final <p> or <table> tag at the end.
  4. Paste the copied text at the end of the original entry file, making a note of the line number. Go back to paste location. Change <div> to <pb>. Delete @type and @xmlns. Remove the final two digits and the hyphen preceding them from the value of @xml:id.
  5. Remove the substitute entry term ending in "CONTD" and validate the corrected file.
  6. Close the continuation file and remove it from the batch by moving it to the ebnn/_removed_files folder for the edition.
