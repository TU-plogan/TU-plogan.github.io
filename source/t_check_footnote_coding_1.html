Check footnote coding 1

Use a simple search to check if you have the right number of note codes on all pages.

Footnotes have two elements:
  1. the footnote "anchor" in the main body of the text, which is usually a superscript number, such as 1.
  2. the footnote "text block," which on a printed page would appear at the bottom, preceded by the superscripted number corresponding to the anchor.
On any single page, there must be exactly the same number of footnote anchors and footnote text blocks. If there are three footnote anchors, there should be three footnotes at the end of the file.
We check for two common problems with footnote formatting in the html page files before converting them into TEI.
  • are there the same number of anchors and text blocks on every page?
  • are all footnote text blocks at the end of the page?
Both checks are done using Oxygen XML Editor to search all 250 files in a section at the same time.
  1. Open the XML-project.
  2. In the Project window, right-click the 2-pages-htmlfolder containing your files, and select Find/Replace in Files.
  3. The Find/Replace in Files window opens. Enter @@ in the Text to find field and click Find all at the bottom of the window.
  4. Matches appear in the Results View, with the search term highlighted.

Test 1: Check for the correct number of footnote anchors and text blocks:

  1. Look at the example above of a page with a single footnote anchor and the corresponding footnote text block. This is typical for every page with one footnote. Why does the same footnote text, with three "@" symbols, appear twice? Because we are searching for two "@" symbols, rather than three, the search counts both the first two and the second two as valid results.
  2. Now look at the line with the file information. It lists the results as "(3 items)" in that file. Every file with one footnote will produce three items. Files with two notes produce six items. If the notes are coded correctly, the number of items in the file should be a multiple of three.

Test 2: Check for the correct placement of footnote text blocks:

  1. The note anchors (<@@>) should all occur first, followed by the note text blocks (<@@@>).
    Click the Location header on the far right of the Find in Files window to sort results by their location in the file.
  2. If notes were not output at the end of the page, you will see <@@@> in the middle of the sequence of <@@> codes. In the following example, the first footnote text block is followed by the second footnote anchor and its text block. It should follow the second anchor, as in the example above.

Testing Procedure:

  1. Go through the search results carefully, checking that (a) the number of items in each file is a multiple of 3; and (b) that all footnote text blocks occur at the end of the list.
  2. Make corrections directly in the HTML file. Resolve extra or missing <@> codes, and move footnote text blocks to the end of the page, in the correct sequence.
  3. Make a note in the inventory file for the section indicating the problem with footnote coding on the page, stating it was fixed in the HTML.
