Document Everything

Document your work on the section inventory file.

The Encyclopedia project contains tens of thousands of individual files, including page images, OCR-Project files, HTML output files, derivative TEI files, and more. It is a multi-year project involving many researchers over time. In addition to a careful scheme of data organization, we maintain careful records of what was done and when. This ensures that nothing is lost when we transition between researchers, and that we have enough documentation to later go back and unravel some mystery.

The primary documentation in the OCR stage is the inventory file, a spreadsheet created for every edition section and its OCR-Project folder. The inventory file documents the production history of that section.

  1. Open a new XSLX (spreadsheet) file and save it as ebnn-rnn-inventory in the home directory for the section it documents.
  2. Create column labels as follows, or copy them from an existing inventory spreadsheet:
  3. In column A, image-fn, enter the original filenames of the images that make up the section. Append the file extension (.tiff, .jp2) to the column label, instead of adding it to every filename. Use the filenames of the images in the production/images directory.
    Excel will populate the list for you if you enter the first 2-4 names, select, and position your cursor over the bottom-right corner of the selection. When the cursor changes to a + sign, left-click and drag down the column to populate it.
  4. Enter the print volume number in column B, p-vol.
  5. Enter the printed page number for each image in column C, p-pg.
    Be sure to account for unnumbered pages, such as plates, even if they were not scanned and output. This gives us a guide to every image file in the section.
  6. Add the initials for the person drawing boxes on each page in column D, scan-by. If one person did the entire section, you only need to enter it once.
  7. Add the date the HTML files were output in column E, output-date.
  8. In column F, output-fn.htm, enter the filename for each HTML file in the output, corresponding to the image filename in column A, image-fn.
  9. Column G, well-formed, indicates when the HTML files are validated in Oxygen XML Editor, as part of the quality-control process.
  10. Column H, notes-prd, documents when the footnote coding, with at codes, is proofed in Oxygen.
  11. We use the next three columns, wd-cnt, errors, and accuracy-rate, to spot-check OCR accuracy on sample pages. We check the total number of words on a page (wd-cnt), and then count the number of words with errors in them (errors). By dividing errors by the word count, we get the accuracy rate for that page.
  12. The final column, comments, is for recording page anomalies. Use this column to note any pages with Hebrew-language text, footnote text of more than one paragraph, or any other information we will need to know for later in the production process.
