Quick check of entry terms

Entry terms in the newly-created entry-inventory should be given a quick check for obvious errors before proceding further. Such a check can reveal problems that can best be solved by modifying the page files and re-running the Python conversion. This generates a new set of files and an updated Entry_Stats.txt file which will then modify the entry-inventory list of terms. If you need to consult the page originals, follow the procedure in Entry-Inventory File.

  1. Review the list of entry terms in the entry-inventory for errors. In addition to known misspelling, check:
    1. Words out of alphabetical order. These are probably scannos.
    2. Words with accidentals. OCR often confuses the descenders from the live above as an accidental for capital letters.
  2. Mark changes in the entry-inventory file first, before fixing the files. This avoids having to make the corrections twice if you discover that you need to rerun the Python conversion.
