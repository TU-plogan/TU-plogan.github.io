Bulk File Renaming

Fixing AFR-generated file names with Bulk Rename Utility

AFR 14 adds spaces within the filenames that we do not want. It also creates a folder containing a single CSS file for each HTML file, appending _files to the same root filename. Finally, it sets up a link between the HTML file and the new folder, so that if you move one, it also moves the other.

To break the link and correct the file names, use a program like Bulk Rename Utility.

  1. Open Bulk Rename Utility.
  2. In the left window, navigate to the folder with the files you want to rename.
  3. Select the HTML files to be renamed in the main window.
  4. In the Replace (3) box, type the portion of the file name you want to change in the Replace field.
  5. In the With field, type the replacement characters.
  6. Preview results in the New Name column in green.
  7. Use the Rename button (not shown) to implement the changes.
  8. After renaming the files, you can move all of the CSS folders into a separate subfolder without disturbing the HTML files. Name the folder css.

Renaming example.

AFR appends a four-digit number to each file name, preceded by - (space-hyphen-space). Note how that portion of the filename and the correction are added to the Replace (3) box, below. A preview of the corrected filename appear in green.
